## Layer 1

Five parallel agents (3 Opus, 2 Haiku) explored naming for the system's approach. Key convergences: Anamnesis (Platonic recollection — learning as remembering what the soul already knows) appeared independently in both Opus agents as top pick. Vigil (keeping watch through the night) resonated with the repo name "night" across all three Opus agents. The ecosystem search revealed a critical positioning gap: all existing systems (Letta/MemGPT, Mem0, Zep, MCP memory servers) treat memory as a tool the agent uses — none frame the knowledge system as the agent's identity itself. SOUL.md is the closest framing but focuses on personality, not persistent operational identity. Key anti-patterns identified: Mem-prefix (exhausted namespace), brain-part names (played out), acronyms (age poorly), mechanism-descriptors (lock to implementation). Strongest candidates by convergence: Vigil, Anamnesis, Palimpsest, Substrate, Eigenself, Sill.

## Layer 2

### Ecosystem Landscape (Haiku agents)

**Existing memory systems — all treat memory as TOOL, not IDENTITY:**
- Letta (formerly MemGPT): "Stateful agents" with OS-inspired memory hierarchy
- Mem0: "Memory layer" with weighted knowledge graphs
- Zep: Temporal/episodic memory with meaning-preserving compression
- Graphiti: Real-time knowledge graphs for continuous integration
- MCP Memory Servers: Knowledge graph via MCP tools — agent CALLS memory, doesn't BECOME it

**The positioning gap:** Every system positions memory as something the agent accesses. None frame the knowledge system as the agent's constitutive identity. The inversion (LLM is the tool, knowledge system is the agent) appears genuinely novel in the landscape.

**Terminology already claimed:**
- "Agent memory," "memory layer," "stateful agents" — generic, overused
- SOUL.md — closest framing (persistent identity in markdown) but focuses on personality
- Engram — taken 6+ times
- Mneme — taken 3+ times
- Mem-anything — exhausted namespace

**Relevant philosophical concepts surfaced:**
- Autopoiesis (Maturana & Varela) — self-producing/self-maintaining systems
- Cognitive Architecture (ACT-R, SOAR) — treats memory as one modular component among others
- "Session-irrelevant" — not an established term; the problem is recognized but unnamed

### Evaluation Framework (Opus 3)

| Criterion | Weight | Rationale |
|-----------|--------|-----------|
| Conveys the inversion | High | Memory-first, session-disposable — the core move |
| Not already claimed | Critical | AI memory namespace is aggressively squatted |
| Project name AND philosophy name | High | "The X approach" and "I'm building X" equally natural |
| Right weight | High | Not cute, not grandiose |
| Night harmony | Medium | Should feel like it belongs with the repo name |
| First-impression legibility | Medium | Directional hint, not full explanation |
| Ages well | High | Still right with vector DB, MCP, multi-project, LLM-interchangeable |
| Short and memorable | Medium | 1-2 syllables ideal, 3 acceptable |

### Anti-patterns identified

1. Mem-prefix family (exhausted, positions as "memory product #47")
2. Greek/Latin neuroscience terms used literally (engram, mneme, hippocampus — played out)
3. Acronyms and backronyms (enterprise feel, age poorly)
4. Mechanism descriptors (lock to implementation — "VectorMind," "KnowledgeGraph")
5. Too dark/edgy (Void, Abyss — try-hard, not constructive)
6. Overclaiming (Omnimind, TotalRecall, PerfectMemory)
7. Too generic (Core, Base, Foundation, Layer)
